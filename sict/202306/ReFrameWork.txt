reframework

트리거로 원하는 시간에 자동화
프로젝트에 투입하게 되면 시퀀스를 안쓰고 reframework를 사용한다.
- 1단계 - init 초기화 단계 
	- asset으로 미리 설정된 초기화설정을 가져온다 (오케스트라를 이용해서 작업?)
	- 초기화도 invoke 활용
	- 필요한것은 미리 config.xlsx파일에 모두 작성
- 2단계 - get transaction data (init성공시)
	- 자동화 진행

- 3단계 - Process Transaction
	- set transaction status 단계에서 구한 에러종류에 따라 로직 분기
	- 예를들어 스크린샷 찍고싶으면 (invokeFile을 사용해서 저장된 워크플로 실행(framework폴더))
	- 3.1단계 set transaction status 단계
		- 에러 발생 여부도 체크 (TryCatch사용)
		- BusinesssError와 SystemError 두개를 만들어서 exception 넣음
	- 3.2단계 Retry transacation? 단계 (재시도방식)
		- 시스템에러면은 재시도를 하고 비즈니스에러면 재시도 종료

- 4단계 - End Process
	- 에러를 모두 처리했거나 없으면 작업 종료

에러 종류
시스템 에러 (창이 튕겼을때)
비즈니스 에러 (예외처리)

에러 발생시
