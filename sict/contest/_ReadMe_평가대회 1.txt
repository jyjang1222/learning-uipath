. 평가대회 관련
	1. 대회 준비
		- 공지
			- 참여자는 집중도 향상을 위해 2000원(인원수에 따라 금액 조정) 이하의 캔 음료수 내기를 한다.(강제 참여 아님)
		- 과제 준비
			- 참여자: 모든 신입사원
			- 과제는 대회 주관자에게 출제 문제 제출한다.(대회 시작하는 날 09시까지)
				- 과제 구성: 다른 신입 사원이 6시간 정도에 풀 수 있는 문제, Workflow로 구성, 실행 동영상(oCam 이용해서 녹화)
				- 과제 종류: 자유과제, 기존 과제와 중복 금지(기존 과제 리스트 제공)
				- 문제 출제 시 반영할 내용
					- 배우지 않는 Activity 사용 시 가이드할 것
					- 문제에 사용할 예제 명시 및 예제 파일 준비
					- 반듯이 따라야 되는 순서가 있다면 기재할 것
				- 준비 기간: 1.5MD ~ 2.0MD
			- UiPath Studio
				모든 참여자의 Version은 같아야 한다.
				Ui는 Classic으로 설정
			
		- 대회 규칙
			- 경기 시작 시간: 10시 00분(참가자 협의로 조절가능)
			- 경기 종료 시간: 17시 00분(미완료 시 출근 전까지 완료)
		
		- 과제 검수 방법
			- 성능 체크는 제출자의 PC에서 진행
			- 출제자 오류로 발생한 추가 작업 시간은 적용하지 않는다.
			- 검수는 당일과 다음 날 시행 2회, 다음날 시행 시 에러 발견되면 미완료자와 동일 순위

		- 벌칙
			- 작업 지시 관련 5번 초과해서 받은 과제 제출자: 과제 완료 작업 시간 + 30분
			- 작업 지시에 없는 내용의 질문을 5번 초과한 수행자: 과제 완료 작업 시간 - 30분
			- 작업 지시에 있는 내용을 질문한 수행자: 질문 금지 30분
			- 인터넷 검색 시 첫 페이지에 검색 결과가 있는 내용 질문한 수행자: 질문 금지 30분
			- 부정 시작 수행자는 대회 시작 전까지 PC 전원 종료, 대회 시작 시 PC 전원 ON 및 과제 완료 작업 시간 + 30분
			- 검색은 핸드폰만 허용: 과제 완료 작업 시간 + 60분
				대기업, 보안 등급 높은 기업, 금융권 프로젝트의 경우 외부 인터넷 사용 금지 되어 있습니다.(사전 체험)
		
		- 회비 기증(내기 참여자 기준)
			각 항목별 당첨자는 전체 회비의 1/3씩을 회비로 기증한다.
			주관자는 아래의 내용을 전체 참여자 기준으로 일감에 등록한다.
			- 가장 먼저 완료한 과제 출제자, 16시전에 완료한 과제 출제자
			- 가장 나중에 완료한 수행자(미완료자가 많을 경우: "전체 회비/3/N"으로 계산)
				- 과제 검수는 당일과 다음 날 시행, 다음날 시행 시 에러 발견되면 미완료자와 동일 순위
				- 17:00까지 미완료한 수행자는 공동 꼴찌
			- Workflow 성능이 수행자 보다 떨어지는 제출자(많은 경우 "전체 회비/3/N"으로 계산)
				성능 점검은 제출자 PC에서 수행
	2. 대회 진행의 예(방식은 자유롭게 진행)
		- 기간: 대회 주관자는 회차별로 (참여 인원수 - 1)일 동안 진행한다.
				- 문제풀이
					- 주관자는 대회 시작 전 문제지를 참여자에게 그룹웨어 메일로 전달
					- 참여자는 브라우저 Close 상태 유지
				- 두번째 대회부터의 문제 질문(첫번째 대회는 제출자에게 직접 질문)
					- 수행자는 문제의 질문을 주관자를 통해서 진행 
					- 주관자는 동일 질문이 있는지 확인 후 출제자에게 대회 끝나고 재수정 지시, 재수정 후 내용 확인
				- 재정비(대회 주관자 전용)
					1. 질문 받은 출제자, 문제 덜 푼 수행자, 먼저 문제 풀린 출제자, 16시전에 완료한 과제의 출제자, 성능 비교에 진 출제자
						-> 대회전 9:30 까지 제출 받을 것
					2. 참여자는 대회 회차별로 완료 후 Redmine 일감의 전체 작업시간 수정
					3. 대회 주관자는 대회 차수별로 결과를 정리한다.(Redmine #1149 참조)
					4. 대회 주관자는 시간안에 문제 해결 못한 수행자에게 어느 부분에서 시간이 걸렸는 지 확인해서 정리한다.
				- 재정비 대상
					재정비를 다음 대회전 9:30 까지 완료 주관자에 제출한다.
					1. 먼저 문제 풀린 출제자, 16시전에 완료한 과제의 출제자
						-> 문제, Workflow 수정
					2. 질문 받은 출제자
						-> 질문 받은 내용을 질문 받지 않도록 문제를 수정.
					3. 성능 비교에 진 출제자
						-> 수행자의 동작 시간보다 3번 동작 평균 10%이상 성능 개선
					4. 해당사항 없는 출제자(Option)
						-> 본인 코드 보완, 문제 수정 불가(대비는 상관없음)
	3. 대회 종료 후
		- 참여자는 자신의 문제 동영상만 보고 PDD 작성, Modern으로 개발: 2MD
